ColdFrame: Reserved names

ColdFrame uses some names to build the framework for you. It will at best cause confusion of you use these names in your models.

The table below shows the names and the circumstances in which it is sensible to use them (on the whole, the names aren't actually ones you would want to use in practice!)

The heading "Class/type?" indicates use as a class or «type». The heading "Attr/opn?" indicates use as an attribute or operation.
