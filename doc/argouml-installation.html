Default settings

Linux and Windows

On Linux and (probably) Windows, you can set up ColdFrame-specific and ColdFrame-recommended defaults in the Edit/Settings menu.

User tab

You can set your name and contact details in the User tab:

Profiles tab

In the Profiles tab, you need to tell ArgoUML where to find ColdFrame's profile (the top half of the screenshot). You'll probably want something more like /where/ever/cf-20121225/lib. Then, in the lower half, click on Refresh List to include ColdFrameProfile.xmi in the Available Profiles and then on the >> to include it in the Default Profiles.

Notations tab

Adjust the Notations settings to suit.

Diagram Appearance tab

In the Diagram Appearance tab, you may like to show node names in a bold font.

Mac OS X

On Mac OS X Lion, with ArgoUML 0.34, the Edit/Settings menu is missing. You can get the effect of the changes above by adding the following lines to your ~/.argouml/argo.user.properties. You'll need to change the values in blue to match where you've installed ColdFrame, and those in green to your own details. 

   argo.notation.default=UML 1.4
   argo.notation.guillemots=true
   argo.notation.hide.bidirectional-arrows=true
   argo.notation.navigation.show.stereotypes=true
   argo.notation.show.associationnames=true
   argo.notation.show.bold.names=true
   argo.notation.show.initialvalue=true
   argo.notation.show.multiplicity=false
   argo.notation.show.properties=false
   argo.notation.show.singularmultiplicities=true
   argo.notation.show.types=true
   argo.notation.show.visibility=true
   argo.profiles.default=CUML 1.4*CGoodPractices*CCodeGeneration*Ufile\:/Users/simon/coldframe/lib/ColdFrameProfile.xmi*
   argo.profiles.directories=/Users/simon/coldframe/lib*
   argo.user.email=simon@pushface.org
   argo.user.fullname=Simon Wright

Project Properties

It's best if you remove all but ColdFrameProfile from the Active Profiles list.
