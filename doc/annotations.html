ColdFrame: Annotations

ColdFrame originally used its own home-grown annotations instead of UML tagged values as a means of decorating models with information to support code generation.

The translation rules are

Annotation Tagged value
[[
{
:
=
,
|
;
,
]]
}

so that the old 

   [[ enumeration : stale, fresh ; callback ]]

becomes 

   { enumeration = stale | fresh, callback }

Annotations had to be in a documentation section, whereas tagged values can be placed anywhere (but it would be a good idea to check out whether ColdFrame actually does what you expect if you want to put the tagged value anywhere else; and remember that too much clutter on a diagram is a Bad Thing).

You could only have one annotation section per documentation section, but multiple semicolon-separated annotations were allowed; you can use as many tagged values as you like, so the example above could equally be written 

   { enumeration = stale | fresh }
   { callback }

ColdFrame used to report all tagged values and unrecognised annotations. Now, any unrecognised tagged values are included in the normalized XML as attributes, so that you can add your own post-processing (for example, special code generation methods).
