ColdFrame: Architecture

The first version of ColdFrame contains a single software architecture. You can look at this from the process view and from the component and runtime view.

Process view

The process involves

  • a target platform,
  • a strategy,
  • preparation of models in Rational Rose,
  • extraction of models from Rose,
  • conversion to standard form,

(which should all be implementation-independent), followed by

  • generation of code
  • according to a set of translation rules,
  • using specially written support components
  • and a set of generic container components (with notes on their use).
  • No doubt you will want to test your code.
  • You may want to generate HTML-formatted documentation.

Component and runtime view

  • Domains
  • Classes, including active classes
    • Attributes
    • Operations
      • Parameters
  • Associations
  • Generalisations/specialisations
  • Types, callbacks and serialization
  • Events
  • Initialization
