ColdFrame: Architecture

The first version of ColdFrame contains a single software architecture. This involves:

  • a target platform,
  • a strategy,
  • preparation of models in Rational Rose,
  • extraction of models from Rose,
  • conversion to standard form,
(which should all be implementation-independent), followed by
  • generation of code
  • according to a set of translation rules,
  • using specially written support components
  • and a set of generic container components (with notes on their use).

I'm considering an alternate presentation of the architecture, with discussion of a number of topics under the headings Motivation (why), Modelling (how, hints and tips, annotations), Translation (mappings, effect of annotations) and Use (tips on calling etc):

  • Domains
  • Classes
    • Attributes
    • Operations
      • Parameters
      • Results
  • Associations
  • Generalisations/specialisations
  • Types
  • Events
